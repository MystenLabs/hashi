armored_encrypt_writer
cert_owns_key_handle
True if cert owns handle — i.e. handle aliases one of the cert’s (primary or sub) keys. Handles the KeyID-vs-Fingerprint suffix matching so a PKESK’s 8-byte KeyID matches a cert key’s full fingerprint.
decrypt_armored_via_gpg
Decrypt an armored OpenPGP message string via gpg --decrypt, returning the plaintext bytes. The ciphertext is piped to gpg over its stdin (a background writer thread) and the plaintext streams back over gpg’s stdout pipe into memory — nothing touches disk.
decrypt_with_gpg
Decrypt backup by shelling out to gpg --decrypt, returning a reader that streams the plaintext.
decrypt_with_gpg_stdin
Like decrypt_with_gpg, but feeds ciphertext to gpg over its stdin instead of a file path. A background writer thread writes the ciphertext so reading decrypted bytes off stdout can proceed concurrently without deadlocking when the ciphertext exceeds the OS pipe buffer (~64 KB on Linux). The thread owns its ChildStdin and drops it on completion, signalling EOF to gpg. The reader joins the writer in Drop so it can’t outlive it.

load_certs
Load and validate each armored OpenPGP cert at paths, logging the fingerprint + path of each. Returns the certs in input order.
pgp_message_recipients
Recipient key handles found in an armored OpenPGP message’s PKESK packets, parsed WITHOUT decrypting. One entry per recipient encryption key. Use to confirm a ciphertext is addressed to the expected cert without holding its secret key (e.g. a yubikey-bound key, which can’t be inspected in memory).
