hashi_types::guardian::log

Module envelope 

Source

The LogRecord envelope written to S3: it wraps a super::message::LogMessage with the session id, timestamp, and (for signed logs) the guardian signature. The object key and lock duration are derived from the wrapped message.

LogRecord
Write: LogMessage -> LogRecord -> JSON body. Read: actual S3 key + JSON body -> untrusted LogRecord -> VerifiedLogRecord.
VerifiedLogRecord
A log record whose message signature and writing session’s attestation/PCRs have both been verified. Its message is normalized to the current schema. If a future schema cannot be converted losslessly, this type should retain VersionedLogMessage and leave version acceptance to callers instead.
