Module log 

Source
Expand description

Guardian S3 logs. message holds the LogMessage family the enclave emits; envelope holds the LogRecord wrapper written to S3 and its keying/signing; ceremony_state combines the ceremony and KP-share messages for readers. The S3 layout constants (stream prefixes + object-lock durations) live here so the whole key/lock scheme reads from one place. See crates/hashi-guardian/README.md for the canonical key layout.

Modules§

ceremony_state
Combined ceremony and KP share state derived from guardian log messages.
envelope
The LogRecord envelope written to S3: it wraps a super::message::LogMessage with the session id, timestamp, and (for signed logs) the guardian signature. The object key and lock duration are derived from the wrapped message.
message
The versioned LogMessage family the enclave emits and its per-message S3 object-key rules. The LogRecord wrapper that carries these to S3 lives in super::envelope.
